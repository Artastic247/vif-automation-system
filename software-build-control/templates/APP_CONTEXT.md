# APP_CONTEXT

## Purpose
Lock app identity and source of truth.

## When to use
Before any review, fix, build, release, or rollout.

## Required fields
- App name
- Purpose
- Users
- Stack
- Repo
- Branch/version
- Environment
- Tenants
- Constraints
- What must not change

## Gate controlled
Context lock.

## Gate decision
PASS / HOLD / BLOCKED
