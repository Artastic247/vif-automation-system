# TURTLE_DIAGRAM

## Purpose
Define process ownership, inputs, outputs, resources, methods, measures, risks, and competence.

## When to use
For each controlled process or workflow.

## Owner/tool
Process owner.

## Inputs
Process scope and evidence needs.

## Activities or fields
Inputs; outputs; with what resources; with whom; how; measures; risks/opportunities.

## Outputs/records
Turtle diagram.

## Gate controlled
No turtle diagram → no process ownership/gap closure.

## PASS/HOLD/BLOCKED rules
PASS if all turtle elements are addressed. HOLD if evidence missing. BLOCKED if owner/resources unknown.

## Update trigger
Process change or audit finding.
