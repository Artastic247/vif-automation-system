# PDCA_REGISTER

## Purpose
Track Plan-Do-Check-Act cycles.

## When to use
For app recovery, process improvement, releases, and corrective actions.

## Owner/tool
Process owner.

## Inputs
Problem/change objective, plan, evidence, result.

## Activities or fields
| ID | Plan | Do | Check | Act | Evidence | Owner | Status |
|---|---|---|---|---|---|---|---|

## Outputs/records
PDCA record.

## Gate controlled
Improvement/control cycle gate.

## PASS/HOLD/BLOCKED rules
PASS if cycle is closed. HOLD if check/action pending. BLOCKED if repeated failure occurs without action.

## Update trigger
Improvement or corrective-action cycle.
