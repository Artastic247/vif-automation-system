# INTERFACE_MAP

## Purpose
Show interfaces between tools, apps, people, data, tenants, and environments.

## When to use
Before integration, automation, release, or data-handling claims.

## Owner/tool
System owner.

## Inputs
Tool list, app list, data flows, API/DB boundaries.

## Activities or fields
Interface; input; output; owner; risk; control; evidence.

## Outputs/records
Interface map.

## Gate controlled
Systems-thinking/interface gate.

## PASS/HOLD/BLOCKED rules
PASS if interfaces and controls are clear. HOLD if partial. BLOCKED if data/security interface is unknown.

## Update trigger
Tool, app, environment, or data-flow change.
