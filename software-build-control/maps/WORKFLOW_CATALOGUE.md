# WORKFLOW_CATALOGUE

## Purpose
Catalogue controlled workflows.

## When to use
Before building or modifying workflows.

## Owner/tool
Workflow owner.

## Inputs
Workflow names, triggers, actors, states, evidence.

## Activities or fields
Defect; change; release; rollout; incident; customer request; maintenance; audit.

## Outputs/records
Workflow catalogue.

## Gate controlled
No workflow map → no workflow build.

## PASS/HOLD/BLOCKED rules
PASS if workflows and owners are defined. HOLD if details missing. BLOCKED if build starts without workflow definition.

## Update trigger
New or changed workflow.
