# SYSTEM_ARCHITECTURE_MAP

## Purpose
Show system layers, repositories, tools, environments, and control boundaries.

## When to use
Before architecture, platform, or integration decisions.

## Owner/tool
System architect.

## Inputs
Repos, tools, environments, control layers.

## Activities or fields
Layer; component; repo; owner; boundary; risk; evidence.

## Outputs/records
System architecture map.

## Gate controlled
Architecture claim gate.

## PASS/HOLD/BLOCKED rules
PASS if boundaries are clear. HOLD if incomplete. BLOCKED if production/control boundaries are unknown.

## Update trigger
Architecture or platform change.
