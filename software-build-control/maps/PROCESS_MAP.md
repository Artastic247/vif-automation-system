# PROCESS_MAP

## Purpose
Map the software build management process from request to lessons learned.

## When to use
Before claiming the operating model is defined.

## Owner/tool
Process owner.

## Inputs
Process scope, roles, gates, outputs, interfaces.

## Activities or fields
Supplier/input/process/output/customer view; sequence; gates; measures; risks.

## Outputs/records
Approved process map.

## Gate controlled
No process map → no operating-model claim.

## PASS/HOLD/BLOCKED rules
PASS if sequence and interfaces are clear. HOLD if gaps exist. BLOCKED if process ownership is unknown.

## Update trigger
Process or interface change.
