# SCREEN_LAYOUT_MAP

## Purpose
Define screen/page layout before UI build.

## When to use
Before UI build, redesign, or Lovable task.

## Owner/tool
Product owner/UI owner.

## Inputs
HTML prototype, screenshots, screen map, workflow, table spec.

## Activities or fields
Page; route; tabs; panels; tables; actions; states; errors; responsive notes.

## Outputs/records
Screen layout map.

## Gate controlled
No screen layout map → no UI build.

## PASS/HOLD/BLOCKED rules
PASS if layout and states are complete. HOLD if gaps. BLOCKED if UI build starts from concept only.

## Update trigger
UI or workflow change.
