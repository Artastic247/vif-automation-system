# PROBLEM_SOLVING_REGISTER

## Purpose
Control structured problem solving.

## When to use
For repeated defects, escapes, serious issues, or systemic failures.

## Owner/tool
Problem owner.

## Inputs
Incident record, evidence, containment status.

## Activities or fields
| Problem ID | Linked incident | Root cause method | Root cause | Correction | Corrective action | Verification | Lessons learned |
|---|---|---|---|---|---|---|---|

## Outputs/records
Problem-solving record and verified corrective action.

## Gate controlled
Problem-solving gate.

## PASS/HOLD/BLOCKED rules
PASS if root cause and CA verified. HOLD if verification pending. BLOCKED if containment missing for high-risk issue.

## Update trigger
Incident, repeat defect, audit finding, or release failure.
