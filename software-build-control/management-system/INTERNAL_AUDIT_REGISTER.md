# INTERNAL_AUDIT_REGISTER

## Purpose
Audit the software build management system and app-control compliance.

## When to use
Scheduled audit or after major failures.

## Owner/tool
Audit owner.

## Inputs
Artefacts, evidence, repo state, release records, incidents.

## Activities or fields
| Audit ID | Scope | Criteria | Evidence sampled | Finding | Risk | Action | Owner | Due date |
|---|---|---|---|---|---|---|---|---|

## Outputs/records
Audit findings and actions.

## Gate controlled
Internal audit gate.

## PASS/HOLD/BLOCKED rules
PASS if audit completed and actions controlled. HOLD if actions open. BLOCKED if critical nonconformity affects release safety.

## Update trigger
Audit schedule or trigger event.
