# LEADERSHIP_GOVERNANCE

## Purpose
Define ownership, authority, decision rights, escalation, release authority, and accountability.

## When to use
Before activating controls, approving job cards, releasing changes, or handling customer-impacting decisions.

## Owner/tool
System owner and release approver.

## Inputs
Roles, responsibilities, approval needs, risk levels.

## Activities or fields
Role; authority; approval rights; escalation path; conflict resolution; release sign-off.

## Outputs/records
Governance matrix and approval evidence.

## Gate controlled
Leadership and authority gate.

## PASS/HOLD/BLOCKED rules
PASS when owners and approvers are known. HOLD if responsibility is unclear. BLOCKED if no accountable owner exists.

## Update trigger
Role, organisation, customer, or release-authority change.
