# STANDARDS_APPLICABILITY_MATRIX

## Purpose
Map standards/frameworks to apps, processes, controls, and evidence.

## When to use
During context analysis, design planning, audit, release, or customer change.

## Owner/tool
Standards owner with human approval.

## Inputs
Standards register, app portfolio, customer requirements, audit needs.

## Activities or fields
Standard/framework name; applicability; app/process affected; source required; configuration required; evidence required; human approval requirement.

## Outputs/records
Approved applicability matrix.

## Gate controlled
Standards configuration gate.

## PASS/HOLD/BLOCKED rules
PASS if mapping is approved. HOLD if applicability unclear. BLOCKED if standard is applied without source or approval.

## Update trigger
Standard/customer/app/process change.
