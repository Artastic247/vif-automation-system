# TOOL_REGISTER

## Purpose
Register tools and approved uses.

## When to use
Before using or adding a tool.

## Owner/tool
System owner.

## Inputs
Tool name, purpose, access, risks, allowed/prohibited uses.

## Activities or fields
| Tool | Purpose | Allowed use | Prohibited use | Access owner | Risk | Status |
|---|---|---|---|---|---|---|

## Outputs/records
Approved tool register.

## Gate controlled
Tool control gate.

## PASS/HOLD/BLOCKED rules
PASS if tool boundaries are clear. HOLD if access/risk unclear. BLOCKED if tool can change protected assets without approval.

## Update trigger
New tool, permission change, or tool failure.
