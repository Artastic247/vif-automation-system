# CAPA_REGISTER

## Purpose
Track CAPA implementation, verification and closure.

## Scope
All corrections, corrective actions and preventive actions linked to software-build NCs and RCA.

## Owner/agent
RCA/CAPA Specialist.

## Inputs
Linked NC/RCA, implementation evidence, verification evidence and effectiveness-review plan.

## Activities/method
Maintain fields: CAPA ID, linked NC/RCA, action type correction/corrective action/preventive action, affected artefacts, owner, due date, implementation evidence, verification evidence, effectiveness-review date, closure decision, status.

## Outputs/records
CAPA register, implementation status and closure evidence.

## Evidence required
Implementation evidence, verification evidence, updated artefacts and closure approval.

## Linked process
Clause 10 CAPA.

## Linked gate
CAPA closure gate.

## MLA / maturity dependency
Ineffective or overdue CAPA can block maturity upgrade and automation permission.

## Clause 9 input
Audit, MLA and validator findings requiring corrective action.

## Clause 10 output
Controlled CAPA closure and effectiveness-review input.

## PASS/HOLD/BLOCKED rules
PASS when CAPA is effective and closed. HOLD when action remains open. BLOCKED when high-risk CAPA is ineffective or unimplemented.

## Escalation
Escalate overdue or ineffective CAPA.

## Management-review input
CAPA trend and overdue status feed management review.

## Continual-improvement input
CAPA lessons feed improvement and knowledge updates.

## Update trigger
CAPA creation, implementation update, verification or closure.
