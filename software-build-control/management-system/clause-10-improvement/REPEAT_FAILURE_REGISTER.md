# REPEAT_FAILURE_REGISTER

## Purpose
Track repeated failure patterns requiring systemic escalation.

## Scope
Repeated software-build, audit, validation, release, security, prompt, AI, workflow and automation failures.

## Owner/agent
RCA/CAPA Specialist with Management Review Owner.

## Inputs
Linked NCs, RCA/CAPAs, audit findings, repeat trends and escalation status.

## Activities/method
Maintain fields: repeat failure ID, failure family, repeated process/module/tool, count/frequency, linked NCs, linked RCA/CAPAs, common cause pattern, escalation status, management-review input, required systemic action.

## Outputs/records
Repeat-failure tracking and systemic-action escalation.

## Evidence required
Failure trend evidence, linked records and escalation evidence.

## Linked process
Clause 10 repeat-failure management.

## Linked gate
Systemic-escalation gate.

## MLA / maturity dependency
Repeat systemic failure can block maturity progression and automation readiness.

## Clause 9 input
Repeated audit and validation findings.

## Clause 10 output
Systemic escalation and improvement requirement.

## PASS/HOLD/BLOCKED rules
PASS when systemic action controls recurrence. HOLD when actions remain open. BLOCKED when recurrence persists without escalation.

## Escalation
Escalate repeated false PASS, failed repair, security risk or ineffective CAPA.

## Management-review input
Repeat-failure trends feed management review.

## Continual-improvement input
Failure families drive strategic improvement priorities.

## Update trigger
Repeat threshold reached, escalation or systemic-action update.
