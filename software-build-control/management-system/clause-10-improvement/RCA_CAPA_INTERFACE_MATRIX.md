# RCA_CAPA_INTERFACE_MATRIX

## Purpose
Define controlled interaction between Clause 9 findings and Clause 10 corrective/improvement activities.

## Scope
Audit findings, MLA assessments, nonconformities, containment, correction, RCA, CAPA, effectiveness review, MLA update, lessons learned, organisational knowledge, prompt update, skill/WI update, agent assignment update, validator/check update, process/procedure update, management review and continual improvement.

## Owner/agent
RCA/CAPA Specialist with Internal Audit Specialist.

## Inputs
Audit findings, MLA decisions, NC records, RCA results, CAPA status and effectiveness reviews.

## Activities/method
Map trigger-to-response flow:
- Clause 9 audit finding -> NC
- MLA failure -> NC/RCA
- False PASS -> escalation/RCA
- NC -> containment/correction
- RCA -> CAPA
- CAPA -> effectiveness review
- Effectiveness review -> MLA update and lessons learned
- Lessons learned -> prompt/skill/validator/process update
- Repeat failure -> management review and continual improvement

## Outputs/records
Interface matrix, traceability route and governance linkage.

## Evidence required
Linked records, approvals, updated artefacts and effectiveness evidence.

## Linked process
Clause 9 and Clause 10 integration.

## Linked gate
Corrective-action governance gate.

## MLA / maturity dependency
Repeated ineffective loops or false PASS conditions block maturity progression and automation readiness.

## Clause 9 input
Audit, MLA, performance-evaluation and validation outputs.

## Clause 10 output
Controlled improvement loop and knowledge update.

## PASS/HOLD/BLOCKED rules
PASS when traceability and closure exist. HOLD when interface links are incomplete. BLOCKED when systemic failures bypass RCA/CAPA.

## Escalation
Escalate broken traceability, repeated ineffective actions or unsupported release decisions.

## Management-review input
Systemic interface failures and repeat themes feed management review.

## Continual-improvement input
Interface effectiveness drives continual-improvement priorities.

## Update trigger
Process change, audit finding, RCA/CAPA update or governance revision.
