# CAPA_CORRECTIVE_ACTION_PROCEDURE

## Purpose
Define correction, corrective action and preventive action controls.

## Scope
All CAPA actions affecting process, agent, skill/WI, prompt, procedure, workflow, validator, MLA record, app/module and tool/provider.

## Owner/agent
RCA/CAPA Specialist with affected owner.

## Inputs
RCA result, linked NC, correction record, affected artefacts and required evidence.

## Activities/method
Define correction, corrective action, preventive action where applicable, affected process, affected agent, affected skill/WI, affected prompt, affected procedure, affected workflow, affected validator/check, affected MLA record, affected app/module, affected tool/provider, action owner, due date, evidence required, effectiveness criteria, closure approval and re-open trigger if ineffective.

## Outputs/records
CAPA action, implementation evidence, verification evidence, effectiveness-review plan and closure decision.

## Evidence required
Implementation evidence, verification evidence, updated artefact evidence and effectiveness criteria.

## Linked process
Clause 10 CAPA.

## Linked gate
CAPA approval and closure gate.

## MLA / maturity dependency
CAPA effectiveness may restore, hold or downgrade maturity.

## Clause 9 input
Audit/MLA/validator/release finding requiring systemic correction.

## Clause 10 output
Implemented action, updated controls and effectiveness-review trigger.

## PASS/HOLD/BLOCKED rules
PASS when action implemented and effectiveness verified. HOLD when implementation/effectiveness pending. BLOCKED when high-risk CAPA lacks owner, evidence or closure approval.

## Escalation
Escalate ineffective CAPA, overdue action or repeated failure.

## Management-review input
CAPA status, overdue actions and effectiveness trends feed management review.

## Continual-improvement input
CAPA outputs update processes, prompts, skills, validators and knowledge.

## Update trigger
RCA output, CAPA implementation, effectiveness review or recurrence.
