# MONITORING_MEASUREMENT_ANALYSIS_EVALUATION

## Purpose
Define what is monitored, measured, analysed and evaluated.

## Scope
Gate performance, validator results, defects, rework, context drift, prompt drift, release evidence, CI pilot, app onboarding readiness, CAPA effectiveness and automation readiness.

## Owner/agent
QA Gatekeeper and Management Review Owner.

## Inputs
Reports, registers, audit findings, incident records, KPI data, validation outputs.

## Activities/method
Define each measure, frequency, owner, threshold, evidence source, action trigger and management-review input.

## Outputs/records
Monitoring record, KPI input, management-review input, improvement trigger.

## Audit criteria
Measurement must use objective records and not assumptions.

## Evidence required
Measurement data, trend review, owner decision, action record where threshold breached.

## MLA / maturity dependency
M4+ requires active KPI and trend review.

## Linked process
Clause 9 monitoring and analysis.

## Linked gate
Monitoring and analysis gate.

## PASS/HOLD/BLOCKED rules
PASS if measures are defined and reviewed. HOLD if data is incomplete. BLOCKED if critical safety/data/release measure is missing.

## Escalation
Negative trend, repeated false PASS, failed rollback, unsafe CI/n8n expansion.

## Management-review input
Trends, thresholds exceeded and blocked gates feed review.

## Update trigger
Management review, new risk, incident, or release.
