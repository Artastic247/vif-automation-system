# MLA_EVIDENCE_MATRIX

## Purpose
Define evidence required for each maturity level.

## Scope
Processes, skills, agents, validators, app modules, automation and onboarding readiness.

## Owner/agent
MLA Assessor.

## Inputs
Maturity definition and controlled item type.

## Activities/method
| Level | Definition | Minimum evidence | Automation permission | Audit need | Gate implication |
|---|---|---|---|---|---|
| M0 | Not defined | None | Blocked | Immediate gap | BLOCKED beyond exploration |
| M1 | Draft | Draft artefact | Blocked | Review required | HOLD unless drafting |
| M2 | Pilot | Controlled pilot evidence | Manual only | Pilot audit | HOLD/PASS by evidence |
| M3 | Released | Approved records | Limited manual use | Scheduled audit | PASS for defined scope |
| M4 | Managed | KPIs, audit, RCA/CAPA active | Limited automation | Periodic audit | PASS with monitoring |
| M5 | Optimised | Measured, audited, improved, reusable | Automation candidate | Risk-based audit | PASS for automation candidate |

## Outputs/records
Evidence matrix.

## Linked process
Clause 9 MLA.

## Linked gate
Maturity gate.

## Evidence required
Per item type and level.

## MLA / maturity dependency
All readiness claims depend on this matrix.

## PASS/HOLD/BLOCKED rules
PASS when evidence meets level. HOLD when partial. BLOCKED when level claimed above evidence.

## Escalation
Maturity overstatement.

## Update trigger
Model change or audit finding.
