# MANAGEMENT_REVIEW_INPUT_REGISTER

## Purpose
Collect required performance-evaluation inputs for management review.

## Scope
Audit results, MLA trends, findings, blocked gates, repeat failures, validator failures, GitHub CI pilot status, n8n readiness, app onboarding readiness, AI/tool audit, prompt/context audit, release/rollback audit, security/data audit, risks, opportunities and improvement recommendations.

## Owner/agent
Management Review Owner.

## Inputs
Audit programme, findings register, MLA register, KPI data, incident/CAPA records, validation reports and readiness records.

## Activities/method
Maintain: input ID, source, summary, risk, affected process, decision required, owner, due date, status and management decision.

## Outputs/records
Management-review input pack and action decisions.

## Audit criteria
Inputs must be current, evidence-based and include blocked gates and critical risk trends.

## Evidence required
Linked record, status, trend or decision need.

## MLA / maturity dependency
Maturity upgrades/downgrades, automation permission and readiness decisions feed review.

## Linked process
Clause 9 management review input.

## Linked gate
Management review readiness gate.

## PASS/HOLD/BLOCKED rules
PASS when critical inputs are captured. HOLD when non-critical inputs are pending. BLOCKED when critical risk, false PASS, unsafe CI/n8n or readiness claim is omitted.

## Escalation
Escalate omitted critical input or unresolved high-risk gate.

## Management-review input
This register is the management-review input source.

## Update trigger
Audit, MLA change, incident, CAPA, validator failure, CI/n8n readiness change, app onboarding request.
