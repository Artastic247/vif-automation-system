# PROCESS_AUDIT_CHECKLIST

## Purpose
Audit process effectiveness, interfaces, controls and runtime evidence.

## Scope
Software-build and management-system processes.

## Owner/agent
Process Audit Specialist.

## Inputs
Process map, WI, runtime evidence, KPI trends, findings and MLA level.

## Activities/method
Check: process owner defined, inputs/outputs clear, interfaces controlled, runtime evidence exists, protected actions enforced, rollback route exists, lessons learned linked, backlog controlled.

## Outputs/records
Completed checklist, findings and audit result.

## Audit criteria
Objective evidence required. Runtime-first evidence required before PASS.

## Evidence required
Process records, runtime traces, approval records, audit trail.

## MLA / maturity dependency
M2+ requires pilot/runtime evidence.

## Linked process
Clause 9 process audit.

## Linked gate
Process audit gate.

## PASS/HOLD/BLOCKED rules
PASS if runtime and control evidence exists. HOLD if evidence partial. BLOCKED if process shell exists without operational proof.

## Escalation
Escalate missing runtime evidence or uncontrolled protected action.

## Management-review input
Critical process gaps and trends feed management review.

## Update trigger
Process change, release, finding or maturity change.
