# MLA_ASSESSMENT_PROCEDURE

## Purpose
Control maturity-level assessment as a Clause 9 performance-evaluation function.

## Scope
Processes, agents, skills, procedures, workflows, prompts, context packs, validators, tools, AI tools, app modules, onboarding, release readiness, GitHub CI readiness and n8n readiness.

## Owner/agent
MLA Assessor with Internal Audit Specialist and process owner.

## Inputs
Artefacts, records, validation reports, audit findings, RCA/CAPA status, management review actions, app evidence.

## Activities/method
Assess item against M0-M5, verify evidence, identify gaps, set audit frequency, determine automation permission, and issue PASS/HOLD/BLOCKED gate decision.

## Outputs/records
MLA assessment record, maturity decision, upgrade/downgrade actions.

## Linked process
Clause 9 performance evaluation.

## Linked gate
MLA gate.

## Evidence required
Current artefact, use evidence, audit evidence, KPI/issue evidence where applicable.

## MLA / maturity dependency
Automation requires maturity permission from MLA matrix.

## PASS/HOLD/BLOCKED rules
PASS if evidence supports maturity. HOLD if evidence is incomplete. BLOCKED if maturity is overstated or automation is requested before maturity permits it.

## Escalation
Escalate maturity overstatement, failed audit, unsafe automation request, or missing owner.

## Update trigger
New process/tool/app, failed audit, RCA/CAPA action, management review, or automation request.
