# COMPETENCE_MATRIX

## Purpose
Control competence for users/tools/roles involved in software build, review, and release.

## When to use
Before assigning tasks or approving releases.

## Owner/tool
System owner.

## Inputs
Roles, tools, required knowledge, evidence.

## Activities or fields
| Role/tool | Required competence | Evidence | Gaps | Action | Approval |
|---|---|---|---|---|---|

## Outputs/records
Competence evidence and gap actions.

## Gate controlled
Support/competence gate.

## PASS/HOLD/BLOCKED rules
PASS if competence evidence exists. HOLD if gaps are minor. BLOCKED if critical release/protected task lacks competence.

## Update trigger
New tool, role, process, or failure.
