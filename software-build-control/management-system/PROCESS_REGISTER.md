# PROCESS_REGISTER.md

## Purpose
Register all controlled processes in the Software Build Control System and link each process to owners, inputs, activities, outputs, records, gates, KPIs, risks, controls, interfaces, skills, and PDCA evidence.

## Scope
All MOP, COP, and SOP processes used for controlled AI-assisted app development.

## Inputs
Process architecture, app-development lifecycle controls, management-system requirements, risk findings, lessons learned, and gate model.

## Activities/checklist
Maintain the process register. No process is released unless it has process ID, owner, inputs, activities, outputs, records, KPI, risk, controls, linked gate, interfaces, and explicit PDCA.

## Outputs/records
Updated process register and linked process matrices.

## Owner/tool
Process Engineer maintains. VIF Orchestrator approves structure. QA Gatekeeper verifies evidence.

## Gate controlled
Process register and process release gate.

## PASS/HOLD/BLOCKED rules
- PASS: all required fields are populated for every process.
- HOLD: process exists but has immature WI, KPI or validation coverage.
- BLOCKED: process lacks owner, PDCA, records, gate or control for protected work.

## Update trigger
New process, process change, audit finding, app failure, tool failure, release issue, or lesson learned.
