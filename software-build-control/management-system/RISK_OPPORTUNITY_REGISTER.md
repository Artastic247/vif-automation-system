# RISK_OPPORTUNITY_REGISTER

## Purpose
Capture risks and opportunities affecting the software build system.

## When to use
Planning, change review, incident review, release, and management review.

## Owner/tool
System owner.

## Inputs
Context, incidents, audits, customer feedback, tool changes.

## Activities or fields
| ID | Risk/opportunity | Cause | Effect | Control | Owner | Status | Review date |
|---|---|---|---|---|---|---|---|

## Outputs/records
Controlled risk/opportunity register.

## Gate controlled
Planning gate.

## PASS/HOLD/BLOCKED rules
PASS if major risks have controls. HOLD if owner/action missing. BLOCKED if critical risk is uncontrolled.

## Update trigger
New risk, issue, change, audit, or release.
