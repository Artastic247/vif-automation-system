# KPI_REGISTER

## Purpose
Track performance of the build management system.

## When to use
Performance evaluation and management review.

## Owner/tool
System owner.

## Inputs
Defects, cycle time, failed checks, release results, rework, incidents.

## Activities or fields
| KPI | Definition | Target | Actual | Trend | Owner | Action |
|---|---|---|---|---|---|---|

## Outputs/records
KPI status and actions.

## Gate controlled
Performance evaluation gate.

## PASS/HOLD/BLOCKED rules
PASS if measured and reviewed. HOLD if trend requires action. BLOCKED if KPI shows critical uncontrolled risk.

## Update trigger
Review cadence or major event.
