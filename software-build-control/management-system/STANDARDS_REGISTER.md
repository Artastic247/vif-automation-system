# STANDARDS_REGISTER

## Purpose
Register standards/frameworks used to shape the management system without embedding proprietary or copyrighted standard text.

## When to use
When a standard/framework affects design, controls, evidence, or customer commitments.

## Owner/tool
Standards owner.

## Inputs
Standard/framework name, applicability, process affected, source required, evidence required.

## Activities or fields
| Standard/framework | Applicability | App/process affected | Source required | Configuration required | Evidence required | Human approval requirement |
|---|---|---|---|---|---|---|

## Outputs/records
Applicability and evidence expectations.

## Gate controlled
Standards applicability gate.

## PASS/HOLD/BLOCKED rules
PASS if applicability and evidence are clear. HOLD if source/evidence missing. BLOCKED if proprietary text is copied or requirement is assumed without source.

## Update trigger
New standard, customer requirement, audit finding, or scope change.
