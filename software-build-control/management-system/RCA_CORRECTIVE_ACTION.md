# RCA_CORRECTIVE_ACTION

## Purpose
Define root cause and corrective action evidence.

## When to use
For systemic defects, repeated failures, or audit findings.

## Owner/tool
Problem owner.

## Inputs
Incident, data, logs, code evidence, process evidence.

## Activities or fields
Problem statement; containment; root cause; correction; corrective action; verification; effectiveness check; lessons learned.

## Outputs/records
Verified RCA/CA record.

## Gate controlled
Corrective-action gate.

## PASS/HOLD/BLOCKED rules
PASS when effectiveness verified. HOLD if evidence pending. BLOCKED if root cause is guessed or action unverified.

## Update trigger
Problem-solving action or audit finding.
