# SYSTEM_CONTEXT_ANALYSIS

## Purpose
Define the context, app portfolio, constraints, interested parties, risks, and operating boundaries of the Software Build Management System.

## When to use
At system setup, major portfolio change, new app onboarding, or strategic change.

## Owner/tool
Product owner with ChatGPT support; repo facts validated by GitHub/Codex.

## Inputs
App portfolio, customer types, standards, tools, environments, tenant model, known risks.

## Activities or fields
Interested parties; internal/external issues; app portfolio; constraints; assumptions; data sensitivity; interfaces.

## Outputs/records
Approved context analysis and open assumptions.

## Gate controlled
System context gate.

## PASS/HOLD/BLOCKED rules
PASS when scope and constraints are clear. HOLD if evidence is partial. BLOCKED if operating boundary is unknown.

## Update trigger
Portfolio, customer, tool, environment, or standard change.
