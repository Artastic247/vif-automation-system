## Use rules

- Check this register before classifying a CI or gate failure as new.
- If a failure is pre-existing, cite the known issue ID in the PR.
- If a persistent blocker is discovered, add it here or raise a follow-up issue.
- Do not use this register to excuse failures introduced by the current PR.
